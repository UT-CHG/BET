Overview¶

Installation¶

The code currently resides at GitHub. If you have a zip file you can install BET using:

python setup.py install

from the package root directory. The BET package is currently NOT avaiable in the Python Package Index this may change in the future. This pacakge requires matplotlib, scipy, mpl_toolkits, numpy, and pyDOE. This package is written in Python.

If you have nose installed you can run tests by typing:

nosetests

in BET to run the serial tests or

mpirun -np NPROC nosetests

to run the parallel tests.

Code Overview¶

calculateP Package¶

This subpackage provides classes and methods for calulating the probability measure \(P_{\Lambda}\).

  • calculateP provides methods for approximating

    probability densities

  • simpleFunP provides methods for creating simple

    function approximations of probability densisties

  • voronoiHistogram provides methods for creating the

    generating points for cells that define a regular grid for use by numpy.histogramdd() and for determining their volumes, etc. This module is only for use by simpleFunP.

  • indicatorFunctions provides methods for creating

    indicator functions for use by various other classes.

sampling Package¶

This subpackage contains

  • basicSampling a general class and associated set of

    methods that interogates a model through an interface.

  • sampler requests data(QoI) at a

    specified set of parameter samples.

  • bet.sampling.adaptiveSampling inherits from

    basicSampling adaptively generates samples.

postProcess Package¶

This subpackage contains

  • plotP plots \(P\) and/or volumes (\(\mu\))

    of voronoi cells

  • plotDomains plots the data domain

    \(\mathcal{D}\) in 2D

  • postTools has tools for postprocessing

sensitivity Package¶

This subpackage provides methods for approximating gradients of QoI maps and choosing optimal QoIs to use in the inverse problem.

  • gradients provides methods for approximating

    gradients of QoI maps.

  • chooseQoIs provides methods for choosing optimal

    QoIs to use in the inverse problem.
